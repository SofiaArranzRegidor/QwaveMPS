# Contributing

Thank you for considering contributing to QwaveMPS!

<!---## Guidelines

To ensure a smooth and collaborative process, please review the guidelines below.

- Fork the repository and create your branch.  
   - Fork the [QwaveMPS repository](https://github.com/SofiaArranzRegidor/QwaveMPS) to your account.
   - Clone your forked repository:
   ```bash
   git clone https://github.com/your-username/QwaveMPS.git
   ```
   - Create a new branch for your changes:
   ```bash
   git checkout -b my-branch
   ```
- Follow the coding standards used in this project.
  - Maintain readability and consistent formatting.
  - Use meaningful commit messages.
  - Write helpful inline comments where necessary.
- Make small, focused commits that address one change or feature at a time.
- Test your changes
  - Run existing tests to ensure nothing breaks.
  - Add new tests if introducing a new feature or fixing a bug.
  - Verify that all tests pass before submitting a pull request:
  ```bash
  # Example: run tests
  make test
  ```
- Open a pull request describing your changes. In your pull request, provide a clear description of the problem you are solving or the feature you are adding.
--->
## Reporting Issues

If you encounter a bug or have a feature request, please open an issue on the [issue tracker](https://github.com/SofiaArranzRegidor/QwaveMPS/issues) to report it. 
- If you encountered a bug, describe it clearly, provide steps to reproduce it and include screenshots and error messages if applicable.  
- If you are requesting a feature, please shortly describe it, describe why it is be necessary or beneficial, and include examples of how it would work.
